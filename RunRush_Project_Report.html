Project Basics

📋 Project Title

RunRush — Your Personal Running Intelligence Platform

📝 Description

RunRush is a full-stack web application designed to help everyday runners track, analyze, and improve their running performance. Users log their runs by entering the date, distance, and duration — and the app automatically calculates pace, calories burned, and generates a personalized AI-style motivational insight based on their recent history. The platform is built around habit-forming mechanics: streaks, badges, weekly goals, and a social feed that lets runners follow each other and stay motivated.

Beyond raw tracking, RunRush functions as a complete running companion. It provides personal bests for 5K and 10K distances, a GitHub-style activity heatmap spanning 365 days, a run-type tagging system (easy, tempo, long, interval, race), weekly email summaries, a community leaderboard, and an admin dashboard for platform moderation. The app works offline — runs logged without internet are saved locally in the browser and synced securely when connectivity is restored.

❓ What Problem It Solves

Most running apps are either too complex (GPS-heavy, subscription-gated) or too simple (just a calorie counter). RunRush hits the sweet spot: a zero-hardware, zero-subscription tracker that provides smart analytics, social motivation, and gamification — helping runners stay consistent through data and community rather than expensive wearables.

Business Logic & Workflows

🏃 Run Logging Workflow

1

User submits Add Run form: date, distance (km), time (min), run type, optional notes

2

Validation: Date ≤ today · distance & time > 0 · pace must be 2–30 min/km (world-record to slow-walk bounds)

3

Calculations: pace = time_min ÷ distance_km · calories = weight_kg × distance_km

4

Insight Generation: analyzes last 30 days, compares pace & distance to personal averages → personalized motivational message

5

Run inserted into runs table with all computed fields

6

user_stats updated: total distance incremented, streak fully recalculated

7

Badge evaluation runs — newly earned badges stored in session; confetti + modal fires on next page load

🔥 Streak Calculation Logic

All unique run dates are fetched and sorted. Current streak: walks backward from today counting consecutive days with a run. Best streak: linear scan of sorted dates finding the longest consecutive chain. Both are recalculated from scratch on every add/delete — no caching risk.

🏅 Badge System

Badge KeyTrigger Condition
FIRST_5KA single run between 4.5–7.0 km
FIRST_10KA single run ≥ 10 km
TOTAL_50KMAll-time distance ≥ 50 km
TOTAL_100KMAll-time distance ≥ 100 km
STREAK_7DAYCurrent streak ≥ 7 consecutive days
STREAK_30DAYCurrent streak ≥ 30 consecutive days

Duplicate prevention via UNIQUE(user_id, badge_key) — once awarded, never duplicated.

📶 Offline Sync Workflow

1

Offline run saved to browser localStorage by sync-engine.js

2

On reconnect, queued runs sent to POST /api/sync-run

3

Server verifies SHA-256 hash of date+distance+time for data integrity

4

Duplicate check: same user + date + distance + time → HTTP 409 Conflict, skipped

5

Stats & badges updated; response includes new badge keys for the client

👥 Social Scoring Algorithm

The Discover Runners section ranks unfollowed users by a dynamic Social Score:

Social Score = (Recent KM × 2) + (Recent Run Count × 5) + (Current Streak × 10)

Where "recent" = last 30 days

This surfaces the most currently active runners — not just historical all-time leaders — making the feed feel alive.

📧 Weekly Email Summary

Sent only to users with (a) a verified email address and (b) opt-in enabled. Uses the Resend API via Python's built-in urllib — no third-party email library needed. The HTML email includes: weekly km, run count, streak, avg pace, goal progress bar, and a personalized motivational message.

Innovation Angle

🤖 AI-Style Contextual Insights

After every run, the app analyzes the user's last 30 days of data and generates a personalized motivational message — comparing today's pace, distance, and consistency to their own history. Zero external AI API cost; pure Python logic.

📊 Engagement-Based Social Scoring

Discover Runners ranks users by: (Recent KM × 2) + (Recent Runs × 5) + (Streak × 10). Surfaces currently active runners — not just historical leaders — making the social graph feel alive and dynamic.

📶 Offline-First Architecture + Hash Integrity

Runs can be logged completely offline. On reconnect, a SHA-256 hash of key fields is verified server-side — tamper-proof offline sync rarely seen in personal fitness apps.

🚫 Zero Hardware Dependency

Deliberately avoids GPS and wearables. Accessible to runners on treadmills, indoor tracks, or without smartwatches. Manual input means 100% privacy — no location data ever collected.

🎮 Gamification Without Pay-walls

Streaks, badges, confetti celebrations, leaderboards — all the motivation mechanics of Strava and Nike Run Club, delivered as a free open app with no subscription or premium tier.

✅ Pace Sanity Validation

Every run is checked against realistic bounds (2–30 min/km). Entries outside this range are rejected with a helpful error — preventing garbage data from polluting leaderboards or insights.

🗄️ Dual-Database Portability

A custom db.py layer supports both SQLite (local dev, zero config) and PostgreSQL (production), with automatic placeholder conversion. Developers run the full app with a single python app.py — no Docker needed.

📅 365-Day Activity Heatmap

A GitHub-style contribution calendar showing a full year of running consistency — a feature typically exclusive to premium running platforms, here built with pure vanilla JS and a simple JSON API.
